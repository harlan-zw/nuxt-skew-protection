## Introduction

Nuxt Skew Protection v1 marks the module's first stable release. The focus: giving you full control over **how** and **when** outdated clients respond to new deployments, while making version state observable on both client and server.

This release also cleans up the API surface, shortening names and aligning conventions before committing to semver stability.

See the [breaking changes](#breaking-changes) below or paste the [LLM upgrade prompt](#llm-upgrade-prompt) into Claude, [ChatGPT](https://chatgpt.com), etc. to migrate automatically.

## 🚀 Features

### Reload Strategies

Previously the module always showed a notification and waited for the user to click reload. Now you choose the behaviour that fits your app with a single config option.

```ts
export default defineNuxtConfig({
  skewProtection: {
    // 'prompt' (default) | 'immediate' | 'idle' | false
    reloadStrategy: 'idle',
  },
})
```

`immediate` reloads the moment chunks are invalidated. `idle` waits until the user switches tabs or the browser is idle, avoiding mid-interaction disruption. Set `false` to disable built-in handling and drive your own logic through hooks.

### Multi-Tab Coordination

When one tab detects a version update, every tab in the same browser now learns about it instantly via `BroadcastChannel`. No duplicate polling, no stale tabs sitting on old code. Enabled by default; disable with `multiTab: false`.

### Health Endpoint

A new `/__skew/health` route returns the current deployment version and server uptime. Useful for load balancer checks, monitoring dashboards, and verifying that a deploy has rolled out.

```json
{ "ok": true, "version": "abc123", "uptime": 3421 }
```

### Server Version Awareness

Every incoming request now carries `event.context.skewVersion` populated from the client's cookie. Use the new `getClientVersion(event)`{lang="ts"} server utility to read it explicitly, or `isClientOutdated(event)`{lang="ts"} to check staleness in API handlers.

```ts
export default defineEventHandler((event) => {
  if (isClientOutdated(event)) {
    // return stale-safe response or set cache headers
  }
})
```

### Rollback Detection

The composable now exposes an `isRollback` computed ref that is `true` when the server version is **older** than the client version, indicating a deployment was rolled back. This lets you distinguish "user needs to update" from "we rolled back, the user is ahead."

### Improved SSR Hydration

`isConnected`, `serverVersion`, and `manifest` now use `useState` internally, ensuring they survive SSR hydration without flicker or double-fetch. The composable also exposes `serverVersion` as a reactive ref so you can display it in admin UIs.

### SkewNotification `isOpen` Slot Prop

The `<SkewNotification>`{lang="html"} component now provides a combined `isOpen` boolean alongside the existing granular `isCurrentChunksOutdated` and `isAppOutdated` props, simplifying the common "show something when anything is stale" pattern.

```vue
<SkewNotification v-slot="{ isOpen, reload, dismiss }">
  <div v-if="isOpen">
    A new version is available.
    <button @click="reload">Reload</button>
    <button @click="dismiss">Later</button>
  </div>
</SkewNotification>
```

## ⚠️ Breaking Changes

### Hook Renamed: `skew:chunks-outdated`

The Nuxt hook `skew-protection:chunks-outdated` has been shortened to `skew:chunks-outdated`.

```ts
// Before
nuxtApp.hooks.hook('skew-protection:chunks-outdated', (payload) => { ... })

// After
nuxtApp.hooks.hook('skew:chunks-outdated', (payload) => { ... })
```

### Composable Return: `isAppOutdated`

`isOutdated` was renamed to `isAppOutdated` to distinguish it from chunk-level staleness.

```ts
// Before
const { isOutdated } = useSkewProtection()

// After
const { isAppOutdated } = useSkewProtection()
```

### Config: `bundleAssets`

`bundlePreviousDeploymentChunks` was renamed to `bundleAssets` for brevity.

```ts
// Before
skewProtection: { bundlePreviousDeploymentChunks: true }

// After
skewProtection: { bundleAssets: true }
```

### Route Prefix: `/__skew/`

Internal API routes moved from `/_skew/` to `/__skew/` (double underscore) to follow the Nuxt convention for internal routes. Update any custom proxy rules or firewall allowlists.

### Cookie Defaults

`sameSite` changed from `strict` to `lax` so the cookie survives cross-origin navigations (e.g. OAuth redirects). `maxAge` shortened from 60 days to 7 days to reduce stale version drift.

### Standalone `checkForUpdates` Removed

The standalone export was removed. Use the composable method instead:

```ts
// Before
import { checkForUpdates } from '#skew-protection'

// After
const { checkForUpdates } = useSkewProtection()
```

### Legacy `_generate` Compat Removed

The internal `_generate` compatibility check from early pre-releases has been removed. If you were on v0.4 or earlier, upgrade to v0.7.6 first, then to v1.
