---
title: Persistent Storage
description: Configure persistent storage for build assets across deployments.
---

To ensure that previous build assets are preserved across deployments, Nuxt Skew Protection requires a storage configuration
that persists data beyond the lifecycle of a single build.

By default, it uses the filesystem storage located at `node_modules/.cache/nuxt/skew-protection`, but this may not be sufficient
for all deployment environments, especially in CI/CD pipelines where caches may be cleared frequently.

## Retention Periods

If needed you can control how long versions are kept, by default the configuration is setup for optimal crawler compatibility.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  skewProtection: {
    // Keep versions for 45 days
    retentionDays: 45,
    // Keep maximum 15 versions
    maxNumberOfVersions: 15
  }
})
```

Cleanup happens **during build**, removing:

- Versions older than `retentionDays`
- Oldest versions beyond `maxNumberOfVersions`

## GitHub Actions

As the default storage is filesystem based, we can leverage GitHub Actions caching to persist the build assets between deployments
using the `actions/cache` GitHub Action.

```yaml [your-github-workflow.yml]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      # Install dependencies first
      # ...

      # Cache skew-protection AFTER install, BEFORE build
      - name: Cache skew-protection
        uses: actions/cache@v4
        with:
          path: node_modules/.cache/nuxt/skew-protection
          key: skew-protection-${{ github.ref_name }}-${{ github.run_id }}
          restore-keys: |
            skew-protection-${{ github.ref_name }}-
            skew-protection-

      # Build, etc..
      # ...
```

## Redis

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  skewProtection: {
    storage: {
      driver: 'redis',
      host: process.env.REDIS_HOST || 'localhost',
      port: Number(process.env.REDIS_PORT) || 6379,
      password: process.env.REDIS_PASSWORD,
      db: 0,
      base: 'skew-protection'
    }
  }
})
```


## Disabling Persistent Storage

In cases where you're already using a CDN with long cache times on `/_nuxt/**` assets, you may want to disable persistent storage.

```ts
export default defineNuxtConfig({
  skewProtection: {
    // Disable persistent storage
    bundlePreviousDeploymentChunks: false
  }
})
```
